As part of my collecitons responsibilities as a liaison librarian, I developed a data-driven process for electronic journal collections evaluation. In addition to the collections assessment using usage statistics, I have been evaluating the collection by looking at the journals researchers publish in and the journals that they cite. Using Python, I developed a tool that could automatically check and see if the journal is in the collection. This was work was presented at the 2019 Canadian Library Assessment Workshop.

    • What happened

    As a liaison librarian at Leddy Library, I am responsible for many things for my departments: outreach, teaching, reference, data and research support, and collections. I am responsible for the monograph and serial collections for biology, biomedicine, chemistry & biochemistry and nursing — including selection, weeding, weighing in on consortial deal renewals, etc. The large number of responsibilities and duties leaves little time for in-depth and proactive collections management.

    A screenshot of a Python script that is automatically checking journal ISSNs against the library OpenURL link resolver.

    Existing evaluation by librarians at UWindsor relied heavily on usage statistics provided by the vendor (when available) and personal judgement. Bibliometric approaches to collections evaluation, like those used in large-scale projects such as the Elsevier cancellation at the University of California, or the CRKN Journal Usage Project, use other metrics to evaluate electronic journals, such as the journals institutional researchers publish in and cite. Such metrics are available in some form from proprietary reports that universities can purchase, such as from 1Science (Elsevier) or from InCites (Clarivate Analytics), but the underlying data is available from the same citation indexes that many universities already subscribe to (e.g., Scopus, Web of Science). In this project, I sought to create these metrics programmatically by using existing citation indexes and open source data sources.

    Using the API for Scopus, I was able to download all the publications by UWindsor biology faculty members. Refining this dataset, I can parse out bibliographic information for the journals that these publications were found in and create a list of authored journals. With the API, I also identified all the referenced sources, and after refining, a list of the cited journals. I wrote a script in Python to do this automatically and utilized the library link resolver to grab information about the library holdings for the journals. As a result, this project enabled me to identify what journals we subscribe to, which ones we don’t subscribe to, and which ones we have embargoed access to — for both journals published in and journals cited.

    So what?

    This information, when used in conjunction with usage statistics and professional judgement, allow us to get deeper insight on the information needs of researchers in these departments. The data analysis techniques allow us to query specific questions and extend the scope of information provided by citation indexes or even the proprietary reports available for purchase. There are some limitations to the information from this report, however.

    First and foremost, the data available to libraries through these APIs is somewhat restricted. Agreements with vendors for the citation indexes limit the type of data and metadata that we can get from the API. For example, the CRKN agreement with Elsevier for Scopus prohibits universities from accessing the complete metadata for references of articles. This means that we can’t get the ISSN, or even a complete title, for the cited journals, which makes the work much more difficult. Of course the data sources are not 100% comprehensive, so further work is also required to merge data from multiple citation indexes. Finally, the program in its current form cannot check print holdings, as they are not included in the link resolver.

    Next steps

    The programs developed in this project are available as open source code on GitHub. The data analysis techniques and process were presented in a workshop at the 2019 Canadian Library Assessment Workshop and are also available openly on GitHub. To date, this information has been used at the time when journal packages are renewed. Looking to the future, I am working to develop a database to sort this data by packages in order to identify overlap of journal coverage between packages, to identify packages for further consideration. Using this information, I am also looking to write regular reports about the collection to faculty members, in order to keep the decision making transparent and to build communication.
